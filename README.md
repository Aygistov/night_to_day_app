# 🌌 Night AI — CycleGAN Night-to-Day Translator ☀️

> **Магістерська кваліфікаційна робота**\
> _Тема: Використання генеративно-змагальних мереж для зміни стилю зображень_

Цей репозиторій містить клієнтський кросплатформний мобільний додаток
(Flutter/Dart), який є практичною реалізацією магістерської роботи. Додаток
призначений для **реальночасової стилізації та покращення нічних зображень до
денного вигляду (Night-to-Day style translation)** за допомогою оптимізованої
генеративно-змагальної мережі **CycleGAN** безпосередньо на кінцевих пристроях
(Edge Inference).

---

## 🚀 Основні особливості проекту

1. **Локальне виконання (Edge AI)**: Усі обчислення виконуються безпосередньо на
   смартфоні чи комп'ютері без надсилання кадрів на хмарні сервери, що гарантує
   100% конфіденційність, автономність та низьку затримку.
2. **Асинхронний пайплайн (Dart Isolates)**: Запуск інференсу моделі у фоновому
   ізоляті (`InferenceWorker`) дозволяє уникнути блокування головного потоку
   інтерфейсу (UI thread). Це забезпечує стабільні 60 FPS для прев'ю камери,
   поки кадр паралельно обробляється нейромережею.
3. **Кросплатформність**: Завдяки Flutter, додаток працює на iOS, Android та
   macOS.
4. **Оптимізований конвертер кольору**: Реалізовано швидкий перехід між YUV420
   (Android), BGRA8888 (iOS/macOS) та RGB тензорами.
5. **Апаратне прискорення**: Інтегрована підтримка GPU delegates (Apple Metal /
   Neural Engine) для сумісних мобільних пристроїв.

---

## 🧠 Нейромережева Архітектура: CycleGAN

В основі проекту лежить архітектура **CycleGAN** (Unpaired Image-to-Image
Translation за допомогою циклічно-узгоджених мереж). На відміну від стандартних
GAN, CycleGAN не потребує спарених зображень «до/після» для навчання, що є
критичним для задач перетворення нічних сцен у денні.

### 📐 Функція втрат (Loss Function)

Математична модель оптимізує комбінацію трьох лосів:

- **Adversarial Loss**: Забезпечує реалістичність згенерованих зображень.
- **Cycle Consistency Loss**: Гарантує, що при зворотньому перетворенні ми
  отримаємо вихідне зображення (запобігає втраті геометрії та семантики):
  $$L_{cyc}(G, F) = \mathbb{E}_{x \sim p_{data}(x)}\left[\|F(G(x)) - x\|_1\right] + \mathbb{E}_{y \sim p_{data}(y)}\left[\|G(F(y)) - y\|_1\right]$$
- **Identity Loss**: Допомагає зберігати оригінальну кольорову гаму сцени, коли
  зображення вже відповідає цільовому домену.

## 🛠️ Оптимізація та Портування Моделі

Оскільки класичні CycleGAN генератори (на базі ResNet-9 або U-Net) є
ресурсомісткими для мобільних платформ, у межах магістерського дослідження
проведено глибоку оптимізацію моделі:

1. **Навчання**: Модель навчалася у PyTorch (`latest_net_G.pth`).
2. **Конвертація в ONNX**: Експорт з PyTorch з оптимізацією графів обчислень.
3. **Конвертація в TensorFlow Lite**: Перехід до TFLite формату
   (`night_vision_optimized.tflite`) з використанням:
   - **Post-Training Quantization (PTQ)**: Зменшення розміру моделі.
   - **Pruning & Weight Hardening**: Видалення зайвих зв'язків без значної
     втрати метрик якості (SSIM/PSNR).
   - **Розмір тензорів**: Вхідний/вихідний розмір становить `256x256x3` з
     нормалізацією значень у діапазон `[-1.0, 1.0]`.

## 🏗️ Схема роботи додатка (Data Flow)

Нижче наведено діаграму обробки кадрів у реальному часі:

```mermaid
graph TD
    A[Камера пристрою] -->|Raw Frame: YUV420 / BGRA| B[Головний потік UI]
    B -->|Надсилання кадру| C[Фоновий Dart Isolate]
    C -->|1. Масштабування до 256x256| D[Передпроцесинг]
    D -->|2. Нормалізація в [-1, 1]| D
    D -->|3. Формування тензора| E[TFLite Interpreter]
    E -->|GPU Delegate / Neural Engine| F[Генератор CycleGAN]
    F -->|Тензор результату| G[Постпроцесинг]
    G -->|1. Денормалізація в [0, 255]| G
    G -->|2. Конвертація в JPEG| G
    G -->|Повернення байтів JPG| B
    B -->|Відображення PiP вікна| H[Користувацький інтерфейс]
```

## 📊 Технічні характеристики

| Параметр                    | Значення                                          |
| :-------------------------- | :------------------------------------------------ |
| **Архітектура Генератора**  | ResNet-based Generator (CycleGAN)                 |
| **Вхідний формат тензора**  | `[1, 256, 256, 3]` (NHWC / float32)               |
| **Вихідний формат тензора** | `[1, 256, 256, 3]` (NHWC / float32)               |
| **Діапазон значень**        | `[-1.0, 1.0]`                                     |
| **Розмір TFLite моделі**    | ~45 MB (базова) / ~7.7 MB (Lite версія)           |
| **Апаратне прискорення**    | Apple Metal Delegate (iOS), XNNPACK (CPU)         |
| **Цільові метрики**         | Максимізація SSIM (Structural Similarity) та PSNR |

## 📂 Структура репозиторію

- `lib/main.dart` — Головний екран камери, UI та керування стрімінгом кадрів.
- `lib/services/inference_worker.dart` — Фоновий ізолят (`InferenceWorker`),
  завантаження моделі з підтримкою GPU-делегатів та конвертації YUV/BGRA.
- `lib/services/model_service.dart` — Додатковий сервіс для прямих маніпуляцій з
  інтерпретатором моделі та буферами.
- `assets/models/` — Збережені оптимізовані TFLite моделі.

## ⚙️ Як запустити локально

### Вимоги:

- Flutter SDK (версія `>= 3.9.0`)
- Встановлений Xcode (для iOS/macOS) або Android Studio з NDK (для Android)

### Кроки:

1. Клонуйте репозиторій:
   ```bash
   git clone https://github.com/your-username/night_to_day_app.git
   cd night_to_day_app
   ```
2. Завантажте залежності:
   ```bash
   flutter pub get
   ```
3. Підключіть фізичний пристрій або запустіть емулятор/симулятор.
4. Запустіть проект:
   ```bash
   flutter run
   ```

> [!NOTE]\
> Для досягнення максимального FPS рекомендується запускати додаток у режимі
> `--release`, оскільки дебаг-режим Flutter має значні оверхеди на логування та
> перевірки типів: `flutter run --release`
